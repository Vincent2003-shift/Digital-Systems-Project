# FPS Cheat Detection — Behavioural Anomaly Detection in Online Multiplayer Games

Dissertation project detecting aimbot-style cheating in FPS matches from raw per-tick
aim/firing telemetry, using both unsupervised anomaly detection (Isolation Forest) and
supervised classifiers (Random Forest, XGBoost), benchmarked against a simple rule-based
baseline.

## Data

This repo contains code only — not the dataset, trained models, or generated outputs
(see `.gitignore`). Data licence on Kaggle is listed as "Unknown", so it is not
redistributed here.

1. Download the dataset from Kaggle:
   https://www.kaggle.com/datasets/emstatsl/csgo-cheating-dataset
2. Place `legit.npy` and `cheaters.npy` in a `data/` folder at the repo root:

   ```
   data/
     legit.npy
     cheaters.npy
   ```

Everything else (`features.csv`, trained models, results, figures) is regenerated by
running the pipeline.

## Install

Requires Python 3.14 (developed and tested on 3.14.3).

```
pip install -r requirements.txt
```

## Run

```
python pipeline.py
```

This runs the full pipeline in order: `features.py` → `train.py` → `evaluate.py` →
`visualise.py`. Each step can also be run individually once its inputs exist.

### Outputs

| Folder      | Contents |
|-------------|----------|
| `data/`     | Raw `.npy` files (you provide) + generated `features.csv` |
| `models/`   | Trained models, scaler, calibrated thresholds, held-out test split |
| `results/`  | Metrics, ROC curves, confusion matrices, error analysis CSVs |
| `figures/`  | Exploratory dissertation-quality plots |

## Files

- `features.py` — extracts per-player behavioural features (aim accuracy, smoothness,
  firing regularity, etc.) from the raw `.npy` engagement data.
- `train.py` — trains the rule-based baseline, Isolation Forest (semi-supervised and
  unsupervised variants), Random Forest, and XGBoost on the extracted features.
- `evaluate.py` — computes metrics (ROC-AUC, precision/recall/F1, recall@1% FPR), plots
  ROC curves and confusion matrices, and runs error analysis (FP/FN inspection) and a
  threshold/use-case table for the best model.
- `visualise.py` — produces exploratory plots (feature distributions, label
  correlations, scatter plots, box plots) from the extracted features.
- `pipeline.py` — runs the four scripts above in order, stopping on the first failure.
- `exploratory/Data.py` — ad-hoc script for dumping the raw arrays to CSV for manual
  inspection; not part of the pipeline and nothing downstream depends on its output.

## Reproducibility

`SEED = 42` is set and used consistently everywhere randomness occurs: the train/test
split, Isolation Forest, Random Forest, XGBoost, and cross-validation folds.
